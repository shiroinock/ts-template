# ステージされた変更からコミットメッセージを生成してコミット

ステージされているファイルのdiffを分析し、適切なコミットメッセージを生成してコミットしてください。

## 手順

1. `git diff --staged` を実行してステージされた変更を確認
2. `git log --oneline -10` で直近のコミットメッセージのスタイルを確認
3. 変更内容を分析し、以下の形式でコミットメッセージを生成：
   - 1行目: 変更の要約（50文字以内、動詞で始める）
   - 空行
   - 本文: 変更の詳細説明（必要に応じて）
4. 生成したコミットメッセージをユーザーに提示し、確認を求める
5. 承認されたら `git commit` を実行

## コミットメッセージのガイドライン

- 日本語で記述する
- 変更の「なぜ」を重視（「何を」はdiffで分かる）
- 関連するIssue番号があれば含める

## 注意事項

- **重要**: ステージされたファイルのみをコミットする。`git add` や `git add -A` は実行しない
- unstagedなファイルやuntrackedファイルには一切触れない
- ステージされた変更がない場合は、その旨をユーザーに伝える
- 機密情報（APIキー、パスワードなど）が含まれていないか確認
- コミット前に必ずユーザーの承認を得る
- コミットコマンドは `git commit -m "..."` のみを使用（`git add` を含めない）
