# プロジェクト構造チェック

## 説明
不用意なディレクトリやファイルが追加されていないか、プロジェクト構造がCLAUDE.mdの仕様通りかを確認します。

**重要**: 許可されたディレクトリ・ファイルの正式な定義は `CLAUDE.md` を参照してください。
要件に従って追加が必要な場合は、まず `CLAUDE.md` を更新してから追加してください。

## 適用条件
以下のいずれかに該当する場合：
- 新しいディレクトリが作成された
- 新しいファイルが追加された
- ファイルが移動・リネームされた

**適用対象**:
- `src/` ディレクトリ以下のソースコードファイル（.ts, .tsx, .js, .jsx）
- テストファイル（.test.ts, .test.tsx）
- スタイルシート（.css, .scss）

**適用対象外**:
- `.claude/` ディレクトリ内のファイル（エージェント、レビューポイント等）
- ルートディレクトリの設定ファイル（package.json, tsconfig.json等）
- ドキュメントファイル（.md）
- その他のメタファイル

## チェック項目

### 基本チェック
- [ ] `src/` 以下のディレクトリが `CLAUDE.md` の「プロジェクト構成」に従っているか
- [ ] 予期しないディレクトリが作成されていないか
- [ ] ファイルが適切なディレクトリに配置されているか
- [ ] 新しいディレクトリを追加する場合、`CLAUDE.md` が先に更新されているか

### ファイル配置ルール（CLAUDE.mdより）
- [ ] 型定義（interface, type）は `src/types/` に集約されているか
- [ ] コンポーネントは `src/components/` 以下の適切なサブディレクトリにあるか
- [ ] ユーティリティ関数は `src/utils/` にあるか
- [ ] カスタムフックは `src/hooks/` にあるか
- [ ] ストアは `src/stores/` にあるか

### 禁止事項
- [ ] `CLAUDE.md` に記載のないディレクトリを作成していないか
- [ ] 一時ファイルやデバッグ用ファイルが残っていないか
- [ ] `.env` や認証情報を含むファイルがコミットされていないか
- [ ] 不要なボイラープレートファイルが残っていないか

### 参照先
プロジェクト構造の正式な定義：
- **CLAUDE.md** の「プロジェクト構成（予定）」セクション
- 新しいディレクトリやファイルパターンを追加する場合は、先にCLAUDE.mdを更新すること

## 重要度
high

## 判定基準

### FAIL
- 仕様にないディレクトリが `src/` 以下に作成された
- ファイルが明らかに間違ったディレクトリに配置された
- 機密情報を含むファイルがコミットされた

### WARN
- 許可されているが推奨されない配置
- 一時的に許容されるがリファクタリングが必要

### PASS
- 全てのファイル・ディレクトリが仕様通りの場所にある
