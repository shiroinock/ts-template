# React コンポーネントレビュー観点

## チェック項目

### 基本構造【必須】
- [ ] **コンポーネントの責務が単一か** → 複数の責務を持つ場合はFAIL
- [ ] **適切な命名規則（PascalCase）** → 違反はWARN
- [ ] **export形式の一貫性** → named/defaultの混在はWARN

### Props とインターフェース【重要】
- [ ] **Props型が定義されているか** → 未定義はFAIL
- [ ] **不要なオプショナルプロパティがないか** → 過度な?使用はWARN
- [ ] **childrenの型が適切か** → React.ReactNodeを推奨

### フック使用【重要】
- [ ] **依存配列の完全性** → 不足はFAIL
- [ ] **不要な再レンダリングの回避** → useMemo/useCallbackの活用
- [ ] **カスタムフックの抽出** → ロジックが複雑な場合はWARN

### 状態管理【重要】
- [ ] **状態の最小化** → 派生可能な状態はWARN
- [ ] **状態更新の安全性** → 直接変更はFAIL
- [ ] **Zustand/Context使用の適切性** → グローバル状態の適切な使用

### レンダリング最適化【推奨】
- [ ] **条件付きレンダリングの効率性** → 早期returnの活用
- [ ] **key属性の適切性** → インデックス使用はWARN
- [ ] **メモ化の適切な使用** → 過度な最適化はWARN

### エラー処理【必須】
- [ ] **エラーバウンダリの考慮** → クリティカルな箇所での欠如はWARN
- [ ] **非同期処理のエラーハンドリング** → try-catchの欠如はFAIL
- [ ] **ローディング/エラー状態の表示** → UXの考慮

### アクセシビリティ【重要】
- [ ] **セマンティックHTMLの使用** → div多用はWARN
- [ ] **適切なARIA属性** → インタラクティブ要素での欠如はWARN
- [ ] **キーボード操作のサポート** → フォーカス管理

### スタイリング【推奨】
- [ ] **CSSクラス名の一貫性** → 命名規則違反はWARN
- [ ] **インラインスタイルの最小化** → 過度な使用はWARN
- [ ] **レスポンシブ対応** → モバイルファーストの考慮

### テスタビリティ【重要】
- [ ] **data-testid属性の付与** → テスト対象要素での欠如はWARN
- [ ] **ロジックの分離** → UIとビジネスロジックの混在はWARN
- [ ] **モックしやすい設計** → 外部依存の抽象化

## 判定基準

### FAIL判定の条件
- Props型が未定義
- 依存配列の不足による無限ループリスク
- 状態の直接変更
- 非同期エラーハンドリングの欠如

### WARN判定の条件
- 複数責務のコンポーネント
- 派生可能な状態の保持
- keyにインデックス使用
- セマンティックHTML未使用
- data-testid属性の欠如

### PASS判定の条件
- すべての必須項目をクリア
- 重要項目で重大な問題なし
- React のベストプラクティスに従っている

## React特有の注意事項

### パフォーマンス
- 不要な再レンダリングの特定
- 大量データ表示での仮想化検討
- 画像・アセットの遅延読み込み

### 保守性
- コンポーネントの適切な分割
- 共通ロジックのカスタムフック化
- 型安全性の確保

### ユーザビリティ
- ローディング状態の明示
- エラー時の適切なフィードバック
- アニメーションの適切な使用
